# Document 9 — Risk Register

Likelihood / Impact each High / Medium / Low. "Validate in phase" references Doc 10 phases.

| ID | Risk | Like. | Impact | Mitigation | Validate in phase |
|----|------|-------|--------|------------|-------------------|
| R1 | **Tool-calling reliability varies across local (Ollama) models** — some emit malformed or text-only tool calls, ignore the tool interface, or loop. | High | High | Per-model capability detection; lenient fallback parser that promotes fenced-JSON tool calls (Doc 6 §6.3); one repair iteration; recommend a known tool-capable model list; the tool-calling harness gate (Doc 10 G1) tests each model before trust. | Phase 3 (harness gate) |
| R2 | **Anthropic tool schema & response divergence from OpenAI** — different tool list key (`input_schema`), tool-call shape (`tool_use` blocks), result feedback (`tool_result` user block), system prompt placement, usage location. | High | Medium | Full normalization in adapters (Doc 4 §4.6); shared `LLMStreamEvent` contract; adapter conformance unit tests run against recorded fixtures for both providers. | Phase 2–3 |
| R3 | **OAuth popup behavior in Edge WebView** — `window.open` blocked/detached; cross-origin postMessage blocked. | Medium | High | Use Office Dialog API + same-origin redirect + `messageParent` (Doc 5 §5.3); explicit Edge WebView popup test gate (Doc 10 G2); fall back to API-key auth if OAuth proves unworkable [D4]. | Phase 8 (gate G2) |
| R4 | **Pricing table staleness** — provider prices change; estimates drift. | High | Low | Versioned `pricing.json` with visible `updatedAt`; default-pricing fallback flagged in UI; in-app editable rates [D7]; cost labelled "estimated". | Phase 6 |
| R5 | **Multi-workbook write targets the wrong workbook** — agent or stale scope writes to the wrong file. | Medium | High | Every tool requires explicit `workbook_id` resolved via registry; ConfirmationModal always labels workbook+sheet; workbook switch blocked mid-run; stale-handle detection auto-rescopes with a notice; degrade to host-only [D8]. | Phase 4–5 |
| R6 | **Chart & pivot Office.js API complexity / version gaps** — properties read-only or unsupported in some Excel builds; pivot API immature. | High | Medium | Allow-list supported chart types/properties; capability detection for pivots; MVP pivot subset (Doc 4 §4.5) [D5]; `Unsupported` ToolResult so the model adapts; coarse structural undo. | Phase 7 |
| R7 | **localStorage quota exhaustion from usage history / snapshots / messages.** | Medium | Medium | Per-day usage buckets + 30-day rolling window; quota-guard wrapper with oldest-first eviction + warning (Doc 7 §7.7); large snapshot/message payloads spill to IndexedDB; snapshot prune policy. | Phase 6 |
| R8 | **HTTPS/mkcert trust failure after Windows updates** — dev cert no longer trusted; pane fails to load. | Medium | High | Startup connectivity/cert self-test with a clear re-trust command (`office-addin-dev-certs install`); document the fix; detect and surface rather than fail silently. | Phase 1 |
| R9 | **Edge WebView loopback exemption reset after Windows updates** — pane can't reach localhost / Ollama. | Medium | Medium | Startup loopback self-test; documented `CheckNetIsolation` exemption; actionable error with the exact command; Ollama-down hint. | Phase 1, 3 |
| R10 | **Context window overflow across a long agentic session.** | Medium | Medium | Token-budget compaction ladder + floor + optional rolling summary (Doc 6 §6.8); live budget gauge; actionable overflow error suggesting larger-context model/smaller scope. | Phase 5 |
| R11 | **Premature/incorrect writes from a misbehaving model** (e.g. fabricated addresses, mass overwrite). | Medium | High | Snapshot-before-write always; mandatory confirmation gate below the loop; per-cell diff; one-click undo; elevated warnings for destructive ops. | Phase 4–5 |
| R12 | **Secret exposure at rest** — API keys/tokens in plaintext localStorage. | Medium | High | Implemented: AES-GCM encryption at rest (`auth/secureStore`), non-extractable key in IndexedDB, legacy plaintext entries migrated on load. Residual: same-origin XSS can still invoke decrypt — OS credential vault via sidecar remains the hardened option; clear-on-signout; no telemetry. | Phase 8 |
| R13 | **Streaming/parse fragility** — reassembling fragmented tool-call JSON; SSE edge cases; abort mid-stream. | Medium | Medium | Robust per-index/blocks reassembly; AbortController cleanup; malformed→repair path; fixture-based tests. | Phase 2–3 |
| R14 | **Snapshot/undo fidelity for structural changes** (charts/pivots/sheet delete). | Medium | Medium | Document `restoreFidelity` levels; coarse undo (delete-created / restore-definition); require explicit confirmation for destructive structural ops. | Phase 7 |
| R15 | **Office.js handle staleness / Excel.run concurrency** — handles invalidated between turns; concurrent runs. | Medium | Medium | ToolExecutor serializes Office.js access; re-resolve handles each call via registry; single active run per session. | Phase 4 |
| R16 | **Sidecar adds attack surface / lifecycle burden** (if introduced). | Low | Medium | Bind 127.0.0.1 only; per-session shared token; own trusted cert; keep optional and behind interfaces [D1]. | Phase 8 |
